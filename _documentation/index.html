Installation

Before you attempt to upload your theme you must unzip the downloaded folder. Why? The extra files it contains can cause strange issues to occur in your theme's functionality. Only upload the inner zip folder and only install your theme through the Wordpress admin installer located under "Appearance" "Themes" "Install".

If this way to install the theme fails, we recommend to use FTP. Open your FTP client and move to your WordPress installation path. Drag & Drop the "theme" folder into the following path: wp-content/themes/ Now go to your "Dashboard" "Appearance" Theme and activate the theme.

We highly recommend to change the permalinks to anything else excerpt default. Otherwise the ajax pagination will not work.

Import Demo Content

To import content go to Dashboard -> DB Settings -> Demo Importer. There are three demos as you can see on our landing page http://wpeventbuilder.com

During the import, the listing/events categories icons and images are not imported.
During the import the MegaMenu settings ar not imported. You have to go to Appearances -> Menus -> Main Menu and make the changes details.

Setup Menu

To setup your own menu, go to Dashboard -> Appearance -> Menus. From top select to create a new menu. Add buttons (pages, categories, links, etc). Select the menu location and save.

MegaMenu

To activate the MegaMenu just expand the menu button and there checkup the megamenu option. It only if the button is level 1.

If you want to have a new column of buttons but with no header title just checkup the "hide title" option.

Gravity Forms Configuration

These settings must set made EXACTLY otherwise the plugin will not function.

In order for the listing author to receive the notification email a few things need to be configured.

Field Settings

The first step is to create a new field on the form you wish to use. We need to create a hidden field. Add the field like a standard Gravity Form field. Once the field is added, please configure to the following settings:

Field Label Listing ID
Default Value {entry_id}

Notification Settings

If you wish to have more than one notification (e.g a notification to the site administrator that this form was used) then you need to configure your first notification with the following settings:

Send To Email
Send To Email no-reply@yoursite.com

These settings must be exact in order to avoid all extra notifications being sent to the listing author.

Now copy the form id and paste it in Dashboard -> WPJobus Options -> Resume Settings/Job Settings/Company Settings. You can create different forms for each of listing type.

Ninja Forms Configuration

In order for the listing author to receive the notification email a few things need to be configured.

Field Settings

The first step is to create a new field on the form you wish to use. We need to create a hidden field. Add the field like a standard Ninja Form field. Once the field is added, please configure to the following settings:

Field Label Listing ID
Default Value Post/Page ID (if available)

Notification Settings

You must create a separate email notification that will be sent to the listing owner. You can use the default Ninja Form email notification but you need to change the "To" field value to the following:

Send To To
Send To Email no-reply@yoursite.com
Now copy the shortcode and paste it in Dashboard -> WPJobus Options -> Resume Settings/Job Settings/Company Settings. You can create different forms for each of listing type.

Contact Form 7 Configuration

Simply activate the Contact Form 7 plugin and you're set! Just remember to assign the form you want to use in "Job Listings > Settings" or "Resumes > Settings"

In the "To" field for "Mail" make sure you enter a valid email address. It does not matter what it is since the listing email address will be used instead; but it needs to be valid. We recommend no-reply@yoursite.com

Now copy the shortcode and paste it in Dashboard -> WPJobus Options -> Resume Settings/Job Settings/Company Settings. You can create different forms for each of listing type.

Tickera Plugin Setup

Tickera is a plugin that allows users to sell tickets on uploaded events. To supoort it you just install the plugin (Go to Dashboard -> Appearance -> Install Plugins).

TicketTailor Plugin Setup

TicketTailor is a plugin that allows users to sell tickets on uploaded events. To supoort it you just install the plugin (Go to Dashboard -> Appearance -> Install Plugins).

No payment is required from you (as the theme / site owner) for the Ticket Tailor plugin. Only the users which want to sell tickets on your site on their event listings using Ticket Tailor paid plans have to pay.

Translation

WPEventBuilder is fully translation ready.

To start the translation we recommend to use the following plugin Codestyling Localization (https://wordpress.org/plugins/codestyling-localization/) to be installed in your Wordpress.

After activating the plugin go to:

  • Tools
  • Localization
  • find: WPEventBuilder - Premium Magazine Theme
  • choose: Add New Language
  • choose Language
  • create po-file
  • do a "rescan"
We constantly develop new features and releasing new Updates for this theme. After downloading the latest Version we recommend to push the "rescan" button and fill out the new items.

Afterwards we recommend to use one of the following plugins to translate your page and posts.

http://wpml.org/ or http://wordpress.org/extend/plugins/qtranslate/
